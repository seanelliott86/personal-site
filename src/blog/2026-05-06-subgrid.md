Subgrid is a game changer and makes your life a hell of a lot easier.\
\
If you inspect my site now your going to see a flatter structure and thats thanks to subgrid.

Previously I defined a reusable grid with multiple named columns. The main idea was so that I could have defined sections like full-bleed, pop-out and one for a my content with a good line-length to help with readability. Very quick, very easy and the idea was reinforced with articles like [Layout Breakouts with CSS Grid - Ryan Mulligan](https://ryanmulligan.dev/blog/layout-breakouts/). Grid is certainly helpful in that way. \
But this immediately showed a problem, for each section I had to reimplement that grid and then apply the desired section like popout.\
Sure I made it reusable with singular classes like \`.wrapper\` for my grid container and \`popout\` or and used the same class to implement the grid each time but it felt wasteful. 

I can define my grid at the top level and subgrid it to align to the columns i want
