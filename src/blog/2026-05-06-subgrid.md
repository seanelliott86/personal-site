Subgrid is a game changer and makes your life a hell of a lot easier.\
\
If you inspect my site now your going to see a much flatter structure and thats thanks to subgrid.

Previously for my design I leaned into using CSS grid. I created a reusable grid with multiple named columns. The main idea by using grid was so that I could have defined sections like full-bleed, pop-out and another for my content so it had a good line-length to help with readability. All my layouts in one place. Very quick, very easy with decisions in one place and the idea was reinforced with articles like [Layout Breakouts with CSS Grid - Ryan Mulligan](https://ryanmulligan.dev/blog/layout-breakouts/). \
But CSS grid immediately revealed a problem. I hadn't used grid before for bigger layouts, I used it here or there when flex box just couldn't cope. But now that I had created a sitewide layout, each time I wanted a new content section I had to reimplement that grid and then apply the desired width I wanted.\
Sure I made my layouts reusable, classes like \`.wrapper\` for my grid container and \`.popout\` or \`.fullone\` – best class name I've ever written btw – for aligning to specific columns in my grid. But that still meant on my homepage \`.wrapper\` was repeated four times. Not earth shattering but it just felt wasteful. 

"Why cant my grid work across everything, why just the direct children of the gird element" I recall grumbling. \
This was in 2022 when in "designed" my site – FYI I'm a frontend developer and a poor designer – and subgrid was only available in Firefox. Life continued on from there with no issues. My site function and no one died because of my repeated use of \`.wrapper\`.

But recently, I had an itch to do more to my site, I had seen subgrid being mentioned in a few newsletters and seen it was baseline, which means I decided to take a look to understand what problems subgrid solve.

Well now I can define my grid at the top level and subgrid it to align to the columns i want
